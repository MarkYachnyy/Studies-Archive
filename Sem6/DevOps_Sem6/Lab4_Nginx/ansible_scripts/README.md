## Генерация RSA-ключа
Для создания зашифрованного RSA-ключа используется команда:

  openssl genrsa -out private.key -aes256 4096

### Параметры:

-out <файл> — сохраняет ключ в указанный файл (например, private.key).

-aes256, -aes192, -aes128 — шифрует ключ с помощью AES (256, 192 или 128 бит). Потребуется ввести пароль.

-des3 — альтернативное шифрование с использованием 3DES.

<размер> — длина ключа в битах (по умолчанию 2048).

## Создание CSR (запроса на сертификат)
Чтобы сгенерировать запрос на подпись сертификата, используется команда:

  openssl req -new -key private.key -out request.csr -subj "/C=RU/CN=example.com"

### Опции:

-new — создает новый CSR.

-key <файл> — указывает приватный ключ для подписи.

-out <файл> — сохраняет CSR в файл (например, request.csr).

-subj "/C=RU/..." — задает данные субъекта (страна, домен и др.) без интерактивного ввода.

## Создание самоподписанного сертификата
Для генерации самоподписанного сертификата на основе CSR используется:

  openssl x509 -req -in request.csr -signkey private.key -out certificate.crt -days 3650

### Ключевые параметры:

-req — указывает, что входной файл является CSR.

-in <файл> — путь к CSR (например, request.csr).

-signkey <файл> — приватный ключ для подписи.

-out <файл> — сохраняет сертификат (например, certificate.crt).

-days <число> — срок действия в днях.

## Подпись сертификата через CA
Чтобы подписать CSR центром сертификации (CA), выполните:

  openssl x509 -req -in request.csr -CA ca.crt -CAkey ca_private.key -out certificate.crt -days 365

### Опции:

-CA ca.crt — сертификат CA (публичная часть).

-CAkey ca_private.key — приватный ключ CA для подписи.
